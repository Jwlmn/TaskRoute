# AGENTS.md

## 项目开发约束（TaskRoute）

### 1. 数据迁移管理规则

1. 开发阶段禁止新增“修补型迁移”来做历史结构调整。
2. 若表结构需要调整，直接修改已有迁移文件，不创建 `update_*`、`delete_*`、`alter_*` 等命名的迁移文件。
3. 迁移文件命名仅允许业务实体创建语义，例如：`create_xxx_table`。
4. 仅在项目进入生产且需要版本兼容时，才允许追加增量迁移。

### 2. 账号与权限规则

1. 系统不开放用户注册功能。
2. 所有账号由管理员在系统内创建、分配、启停和重置。
3. 默认角色划分：
   - `admin`：全权限
   - `dispatcher`：调度相关权限
   - `driver`：仅个人任务权限

### 3. 登录安全规则

1. 登录页必须启用图片验证码。
2. 验证码失败时拒绝登录并提示重新获取。
3. 验证码需有过期时间且一次性使用。

### 4. 模拟数据要求

1. 开发环境必须提供可直接联调的模拟数据（用户、车辆、货品分类、禁混规则、预计划单）。
2. 模拟数据需覆盖油品与商超两个典型场景，便于验证禁混与派单策略。

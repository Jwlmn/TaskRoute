# TaskRoute 中文智能调度平台需求说明

## 一、项目定位

TaskRoute 是一个面向多行业运输与配送场景的智能派单平台，适用于化工、石油、商超、商场、冷链、仓配等业务。平台以“预计划单 + 智能调度 + 人工可干预 + 全程可视化执行”为核心，帮助企业提升车辆利用率、降低调度成本、减少人工经验依赖，并确保运输过程合规、安全、可追溯。

平台采用统一权限体系下的双终端架构：PC 端与移动端共享同一账号与权限模型。系统不再按 B/C 业务身份分站拆分能力，而是按角色（管理员、调度员、司机）动态分配可见功能。前端需同时支持桌面与移动访问，确保管理员与调度员在移动端同样可进行必要操作。

## 二、建设目标

本项目目标是建设一个可跨行业复用的智能调度中台，重点实现以下能力：

1. 支持企业在系统内维护预计划单，并基于计划单进行自动派单与拼单。
2. 支持单车单订单、单车多订单、多车单订单、多车多订单等多种调度与拼单模式。
3. 接入高德 API，对线路、限行、里程、油耗等因素进行优化，提升配送效率与成本控制能力。
4. 实现调度结果可人工二次调整，避免“全自动不可控”问题。
5. 实现司机实时定位、轨迹监控、任务节点执行与电子单据上传，形成完整闭环。
6. 通过货物属性与车辆能力约束，杜绝混装、错装、违规运输。
7. 建立清晰的权限体系，确保管理员、调度员、司机各司其职、权限隔离。

## 三、适用行业场景

平台具备行业通用性，同时支持行业规则扩展。

典型场景包括：

1. 石油/化工运输
   例如中石化油品配送场景中，运输汽油的车辆不得运输柴油，不同危险品类别不可混装，需结合车辆资质、罐仓属性、限行规则进行调度。
2. 商超/零售配送
   例如超市配送中，运输海鲜的车辆不能与蔬菜、水果等普通货品混装，需结合冷链条件、温区要求、配送时效进行智能拼单。
3. 商场/仓配场景
   支持门店补货、定时配送、多站点联配、波次任务分发、到店签收与异常回传。

## 四、系统建设范围

### 一期：统一权限门户

一期重点完成企业内部运营闭环，包含：

1. 基础资料管理
2. 订单/预计划单管理
3. 智能派单与手动派单
4. 拼单与拆单管理
5. 路线优化
6. 车辆与司机任务执行
7. 实时定位与轨迹监控
8. 电子单据回传
9. 权限与组织管理
10. 报表与运营分析

### 二期：外部协同扩展

在一期基础上扩展：

1. 外部客户协同入口
2. 客户下单与查单
3. 配送进度可视化
4. 签收确认
5. 服务评价
6. 对账与结算入口

## 五、核心用户角色

1. 管理员
   拥有系统全部权限，可管理组织、角色、基础配置、车辆、司机、客户、业务规则、权限、系统参数等。
2. 调度员
   可操作智能派单、手动派单、修改未完成订单、调整拼单结果、查看车辆和司机状态、处理异常任务。
3. 司机
   仅可查看与自己相关的任务订单，可接收任务、执行任务、上报节点状态、上传电子单据、查看导航与线路。

## 六、核心业务流程

1. 企业在系统中维护预计划单。
2. 系统基于订单信息、车辆资源、司机状态、货物属性、配送时效、区域规则进行智能匹配。
3. 系统生成派单结果与拼单建议。
4. 调度员可对智能结果进行人工二次修改。
5. 任务下发至司机手机端。
6. 司机按任务节点执行装货、发车、到站、交付、回单等操作。
7. 司机上传电子单据、签收凭证、现场照片等资料。
8. B 端实时查看司机位置、轨迹、任务状态与异常情况。
9. 任务完成后进入统计分析、对账、归档流程。

## 七、核心功能需求

### 1. 基础资料管理

1. 客户管理
2. 收发货地址管理
3. 车辆管理
4. 司机管理
5. 货物/品类管理
6. 车辆资质与可承运范围管理
7. 行业规则与混装禁忌规则配置
8. 区域、线路、站点、仓库、门店管理

### 2. 预计划单管理

1. 支持系统内人工录入预计划单
2. 支持批量导入预计划单
3. 支持计划单状态管理
4. 支持按日期、区域、客户、货物类型、优先级筛选
5. 支持计划单拆分、合并、作废、锁定
6. 支持生成调度候选池

### 3. 智能派单与拼单

1. 支持单车单订单
2. 支持单车多订单
3. 支持多车单订单
4. 支持多车多订单
5. 支持按线路、区域、时间窗、车型、载重、载积、货物属性、禁混规则进行智能匹配
6. 支持多订单拼单建议与多车拆单建议
7. 支持自动推荐最优车辆与司机
8. 支持人工二次调整、锁单、改派、撤回、重新计算

### 4. 路线优化

1. 接入高德 API 进行线路规划
2. 支持综合考虑限行、时效、里程、油耗、通行成本
3. 支持多站点配送顺序优化
4. 支持展示预计里程、预计耗时、预计油耗
5. 支持调度员手动调整停靠顺序后重新计算路线

### 5. 车辆与司机执行管理

1. 司机手机端接收任务
2. 支持任务确认、拒单申请、异常上报
3. 支持节点化执行，例如到仓、装货、出发、到达、卸货、签收、回单
4. 支持上传电子单据、签收图片、现场照片、异常凭证
5. 支持司机查看配送路线与任务详情

### 6. 实时定位与轨迹监控

1. 管理员与调度员实时查看司机当前位置
2. 支持查看车辆历史轨迹
3. 支持任务地图监控
4. 支持偏航、长时间停留、超时未达等异常提醒
5. 支持按司机、车辆、任务维度追踪执行情况

### 7. 混装与合规控制

这是平台的核心业务规则之一，必须作为系统底层能力建设：

1. 支持配置货物 incompatibility 规则
2. 支持配置车辆承运限制规则
3. 支持按货物品类、危险等级、温区、污染风险、气味交叉风险等维度限制混装
4. 当调度结果违反规则时，系统必须阻止派单或给出强提示
5. 典型例子：
   运输汽油的车辆不可承运柴油
   运输海鲜的车辆不可与蔬菜拼装
6. 支持行业专属规则扩展，便于后续适配更多行业

### 8. 异常与人工干预

1. 所有智能派出的订单均可人工二次修改
2. 支持改派司机、换车、拆单、并单、调整顺序
3. 支持任务中途异常处理，例如车辆故障、司机请假、客户拒收、地址变更
4. 支持异常任务重新调度

### 9. 权限与组织管理

1. 管理员拥有全部权限
2. 调度员拥有智能派单、手动派单、修改未完成订单、监控任务、处理异常等权限
3. 司机只拥有查看本人相关订单与执行任务权限
4. 支持角色、菜单、操作按钮、数据范围权限控制
5. 支持按组织、站点、区域做数据隔离

### 10. 统计分析

1. 派单成功率
2. 车辆利用率
3. 订单准时率
4. 司机履约率
5. 异常任务统计
6. 线路成本分析
7. 油耗与里程分析
8. 回单及时率分析

## 八、关键业务规则建议

为保证系统真正可用，建议在需求中明确以下规则优先级：

1. 安全合规约束优先于成本优化
2. 货物禁混规则优先于拼单收益
3. 时效承诺优先于车辆满载率
4. 人工锁定任务不可被自动算法覆盖
5. 已开始执行的订单不可随意重派，必须经过异常流程
6. 司机、车辆、货物、站点四类约束共同参与调度决策

## 九、订单与任务状态建议

建议建立统一状态流转，便于后续开发与统计：

1. 预计划单
   待创建、待调度、已调度、执行中、已完成、已取消
2. 派车任务
   待下发、待司机确认、待执行、执行中、异常中、已完成、已关闭
3. 回单状态
   待上传、已上传、待审核、已归档

## 十、技术架构建议

### 后端

1. `Laravel 12`
2. `PostgreSQL`
3. `Redis`
4. 数据库与缓存采用 Docker 部署，镜像版本使用 `latest`

### 前端

1. PC 端管理门户：`Vue 3 + Element Plus`
2. 移动端门户：`Vue 3 + Element Plus`（手机优先）
3. 两端共享同一账号与权限体系，按角色展示功能
4. 管理员/调度员/司机均可在移动端使用授权功能
5. 项目整体采用 `Element Plus` 默认主题风格，主视觉以蓝白配色为主，保持简洁、标准、易用的企业级界面风格

### 建议补充

1. 地图服务：高德地图 API
2. 消息通知：站内消息 + 短信/微信可扩展
3. 文件存储：电子单据、图片、签收附件需要对象存储方案
4. 队列机制：用于定位上报、轨迹处理、派单计算、消息推送
5. 日志审计：记录派单、改派、取消、异常处理全过程

## 十三、项目正式介绍版本

TaskRoute 中文智能调度平台，是一套面向化工、石油、商超、商场等多行业运输与配送场景的智能派单与执行管理系统。平台基于 Laravel 12、Vue 3、Element Plus、PostgreSQL、Redis 构建，采用 PC + 移动双终端模式，并统一账号与权限体系。系统通过角色权限控制不同用户可见能力，确保两端能力一致且可管控。

平台以预计划单管理为起点，支持单车单订单、单车多订单、多车单订单、多车多订单等多种派单与拼单模式，并可结合高德 API 对限行、里程、油耗、线路顺序等因素进行优化计算，提升调度效率与运输效益。系统支持所有智能派单结果由调度员进行人工二次调整，确保自动化能力与业务灵活性兼顾。

在执行层面，管理员与调度员可实时查看司机定位与历史轨迹，司机通过移动端接收任务并按节点执行，在装货、运输、交付、签收等关键环节上传电子单据和现场凭证，形成完整业务闭环。系统同时具备严格的货物与车辆匹配规则，可针对油品运输、冷链配送、商超补货等场景配置禁混装规则，杜绝不合规运输行为。结合管理员、调度员、司机三级权限体系，TaskRoute 能够帮助企业实现运输调度标准化、可视化、智能化与可追溯化。
